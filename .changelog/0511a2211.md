## API changes in this session

### Zone Service (backend)
- Added GET `/api/v1/addresses/by-point`
  - Query: `lat`, `lon`, `radius?` (default 75, max 100), `limit?`
  - Response: `{ result: { local: NearestAddressResult[], external: { placeId, name, formattedAddress?, lat, lon, types? }[] } }`
- Added GET `/api/v1/addresses/search`
  - Query: `q`, `limit?`
  - Response: `{ result: { local: AddressDto[], external: { placeId, name, formattedAddress?, lat, lon, types? }[] } }`
- Create enrichments:
  - `POST /api/v1/addresses` now enriches `addressText`, `wardName`, `districtName` via TrackAsia Nearby when missing.
- Behavior tweaks:
  - By-point default radius increased to 75m (cap 100m); returns both `local` and `external` together.

### API Gateway
- Proxies for Zone Service endpoints:
  - GET `/api/v1/addresses/by-point` → Zone Service
  - GET `/api/v1/addresses/search` → Zone Service

### TrackAsia integration (server-side)
- New client calls (internal):
  - Nearby Search: `GET https://maps.track-asia.com/api/v2/place/nearbysearch/json?location={lat},{lon}&radius={r}&key=...`
  - Text Search: `GET https://maps.track-asia.com/api/v2/place/textsearch/json?query={q}&key=...`
- Env: `TRACKASIA_API_KEY` (or `TRACK_ASIA_API_KEY`).

### Frontend usage
- Through gateway base `/api/v1`:
  - By point: `GET /addresses/by-point?lat=..&lon=..&radius=75&limit=10`
  - Search: `GET /addresses/search?q=...&limit=10`
  - Create: `POST /addresses { name, addressText?, lat, lon }`
  - Update: `PUT /addresses/{id} { name?, addressText?, lat?, lon? }`
  - Get by ID: `GET /addresses/{id}`
  - Delete: `DELETE /addresses/{id}`

### Notes
- Responses conform to BaseResponse per RESTFUL.md (`{ result, message? }`).
- By-point endpoint returns both internal (local) and external (TrackAsia) in one call.
